# 基于稀疏采样的可证安全生成式隐写系统

基于稀疏采样（SparSamp）算法的面向生成式模型的可证安全隐写系统，通过介入大语言模型（LLM）推理阶段的随机采样过程，在保障输出文本与原生模型分布统计不可区分的前提下，实现了秘密信息的高效嵌入与完美还原。

---

## 🏗️ 系统总体架构

系统采用前后端分离架构，确保隐写逻辑与模型推理逻辑的解耦。

```text
┌────────────────────┐       ┌────────────────────┐       ┌────────────────────┐
│  数据交互层 (Web)   │       │  隐写调度层 (API)  │       │ 模型引擎层 (LLMs)  │
│ (Vue3 + Tailwind)  │ ────▶ │  (Flask Backend)   │ ────▶ │ (GPT-2 / Qwen-2.5) │
└────────────────────┘       └────────────────────┘       └────────────────────┘
```

| 组件 | 技术栈 | 核心职责 |
| :--- | :--- | :--- |
| **Frontend** | Vue.js 3, Vite, Tailwind | 用户交互、动态结果展示、信息提取验证 |
| **Backend** | Python, Flask, PyTorch | 算法调度、参数管理、API 封装 |
| **Algorithm** | SparSamp Core | 概率分布计算、稀疏采样映射、信息流转换 |
| **Models** | GPT-2, Qwen-2.5 | 提供词表概率分布 (Logits) |

## 💡 设计理念 (Design Philosophy)

* **可证安全为核心**：坚持信息论意义上的可证安全，拒绝任何会破坏模型原生分布的采样修改，理论上实现 **$D_{KL}(P_{model} || P_{stego}) \to 0$**。
* **计算效率优先**：算法复杂度为 $O(1)$，不随词表规模（Vocab Size）增加而产生计算负担。
* **跨模态泛化设计**：隐写层作为“**模型无关（Model-agnostic）**”的原语存在，无需调整模型内部结构即可无缝迁移至图像（DDPM）、音频（WaveRNN）等生成模型。
* **低耦合架构**：前端负责逻辑展示，后端负责算法调度，模型引擎独立运行，确保系统具备极强的扩展性与维护性。

## ✨ 功能特性

### 隐写引擎服务
* **信息嵌入**：将秘密消息精确调制到 LLM 生成的自然文本流中。
* **信息提取**：从隐写文本中 **100% 还原**原始二进制比特流，确保通信可靠性。
* **安全性监测**：实时监测生成文本与原生分布的 **KL 散度**，确保统计不可区分性。
* **多模型驱动**：基于 SparSamp 算法，支持文本（GPT/Llama）、图像（DDPM）、音频（WaveRNN）等多模态生成式模型。

### 交互与通信
* **Web 端交互**：基于 **Vue3** 的可视化操作界面，支持消息实时生成、比对与验证。
* **API 接口化**：后端提供标准 **RESTful 接口**，支持第三方应用接入隐写调度能力。
* **参数动态配置**：支持自定义随机种子（Seed）、嵌入块长度（Lm）及生成长度限制。
* **全链路监控**：展示从 “比特流 $\to$ 概率空间 $\to$ Token 映射” 的完整逻辑路径。

## 📡 API 端点说明

系统后端通过标准 RESTful 接口对外提供隐写服务：

| 方法 | 路径 | 参数 | 说明 |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/stego/embed` | `message`, `model_type`, `seed` | 秘密信息嵌入，返回生成的隐写文本 |
| `POST` | `/api/v1/stego/extract` | `stego_text`, `model_type`, `seed` | 从隐写文本中逆向提取原始信息 |
| `GET` | `/api/v1/model/status` | - | 获取当前模型引擎状态与显存占用情况 |

## 🔄 隐写通信流程



### 1. 发送端 (Alice) 嵌入流程
1. **比特转换**：将秘密信息映射为二进制流。
2. **分布预测**：根据上下文，通过模型获取下一步所有词的预测概率。
3. **稀疏采样**：SparSamp 算法根据当前比特位在候选空间中选定 Token。

### 2. 接收端 (Bob) 提取流程
1. **状态对齐**：使用共享密钥（Shared Seed）复现相同的预测分布。
2. **逆向推导**：识别隐写文本中的 Token，推导出对应的原始比特位。
3. **信息还原**：拼接比特流，恢复原始秘密消息。

## 🚀 快速开始

### 1. 启动后端 (Server)
```bash
cd backend
pip install -r requirements.txt
python app.py
```

### 2. 启动前端 (Web)
```bash
cd frontend
npm install
npm run dev
```
